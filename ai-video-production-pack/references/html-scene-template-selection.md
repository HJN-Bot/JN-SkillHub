# HTML Scene Template Selection for Video

Use when a video needs HTML/Remotion-style scenes, slide-like motion graphics, title cards, architecture diagrams, dashboards, timelines, or high-density explanation screens.

Source repo:
- Local: `/Users/jianan/.openclaw/workspace/repos/beautiful-html-templates`
- GitHub: `https://github.com/zarazhangrui/beautiful-html-templates`

## Core rule

Use `beautiful-html-templates` as a visual-system reference, not as a content-density ceiling.

For Jianan's PLO/CER/AIPM/sharing videos:
- Keep high-density content logic: frameworks, workflow maps, architecture maps, evidence, risks, ROI, and before/after comparisons.
- Borrow color, typography, slide grammar, title-card energy, card systems, grids, and spacing.
- Convert dense content into animated sequences, not empty aesthetic slides.

## Recommended visual systems

- `blue-professional`: credible B2B internal sharing, clean and trustworthy.
- `cobalt-grid`: AI system architecture, data pipeline, dashboard, node/agent maps.
- `neo-grid-bold`: opening hook, strong claim, before/after contrast.
- `long-table`: timeline, task board, roadmap, comparison, project-management sections.
- `monochrome`: serious report / evidence / audit trail scenes.

## Video usage pattern

1. Pick one primary visual system for the whole video.
2. Build 2–3 title/scene previews if the style is not locked.
3. Use HTML/Remotion for exact text, UI, tables, architecture diagrams, and screenshots.
4. Use AI image/video only for atmosphere, metaphors, transitions, and B-roll.
5. Preserve light-tech internal-sharing constraints unless the lane asks otherwise.

## Negative constraints

- Do not drift into dark neon/purple cyberpunk for internal sharing.
- Do not generate unreadable UI or exact Chinese text with AI video models.
- Do not mix visual systems from multiple templates in one scene sequence.
- Do not reduce AIPM/CER/PLO explanations to low-density slogan cards.
