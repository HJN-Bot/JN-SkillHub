# 3 种教导输出方式

每章选一种主导模式。同一本书的不同章可以用不同模式。

## 模式 A · First Principles（从底层概念建起）

### 适用场景
- 方法论的核心章节
- 原理性内容（"为什么这样做"）
- 想让读者推导出结论而非接受结论

### 章节结构

```
1. 起点（Premise）
   - 一个最简单、最难反对的事实或直觉
   - 例："LLM 的输出会变，但原文不该变"

2. 推导（Derivation）
   - 加 1-2 个原则
   - 例：原则 1：所有变化必须可观察 / 原则 2：失败必须可降级
   - 推出更复杂的结论：Anchor Verify 三级降级

3. 验证（Verification）
   - 用一个具体例子证明推导成立

4. 拓展（Extension）
   - 这个推导还能解释什么现象

5. 收口（Crystallization）
   - 浓缩成一句口诀 / 公式
   - 例："不追求 100% 准确率，追求 graceful failure"
```

### 文风特征
- 短句、逻辑连接词密集（因此、所以、那么、可见）
- 不用煽情
- 多用反问把读者拉进推导

---

## 模式 B · Case-First（先讲具体案例再抽象）

### 适用场景
- 经验沉淀章节（"我们踩过的坑"）
- 实战教学（"怎么做"）
- 想让读者先共情场景再接受抽象

### 章节结构

```
1. 现场（Scene）
   - 把读者拉到一个具体时空
   - 例："2026 年 4 月 3 日，我们的 pipeline 第 N 次崩了"

2. 困境（Predicament）
   - 当时遇到的具体问题
   - 例："Normalize 一加，Marker 就乱"

3. 错误尝试（Failed Attempts）
   - 我们先试了什么、为什么不行
   - 例："试过补丁函数，越补越乱"

4. 转折（Turning Point）
   - 什么让我们想到真正的解法
   - 例："想起 Unix pipe 的哲学"

5. 抽象（Abstraction）
   - 从这个案例抽出可复用的原则
   - 例："五段分离 + 中间态物化"

6. 反例（Counter）
   - 这个原则什么时候也会失效
   - 例："如果 stage 之间是高频交互，物化反而是负担"
```

### 文风特征
- 第一段必须有具体时间地点人物
- 多用"我们当时...""那一刻..."
- 抽象段要短，不要喧宾夺主

---

## 模式 C · Dialogue（保留对话张力）

### 适用场景
- 争议性话题
- 想展示思考过程而非结论
- 想保留多种声音

### 章节结构

```
1. 开场（Provocation）
   - 抛出一个有真实争议的问题
   - 例："AI 该不该追求 100% 准确率？"

2. 对话推进（Exchange）
   - A 说 → B 说 → A 反驳 → B 修正
   - 每轮都让立场更精准
   - 不要预设答案

3. 关键转折（Pivot）
   - 某个反例 / 类比让对话进入新阶段
   - 例："医生从来不追求 100% 诊断准确，他们追求'误诊后果可控'"

4. 共识（Convergence）
   - 双方能达成什么部分

5. 悬置（Suspension）
   - 故意保留的开放问题
   - 让读者自己接着想
```

### 文风特征
- 对话格式：可以用 `**A:**` `**B:**` 或者命名两个角色
- A 和 B 的立场要真，不要稻草人
- 每轮发言不超过 3-4 句

---

## 模式选择速查

| 章节内容 | 推荐模式 |
|---------|---------|
| Anchor Verify 为什么要三级降级 | First Principles |
| 我们怎么从 1 月困境走到 4 月成果 | Case-First |
| AI 该不该替代 SME | Dialogue |
| Harness 五段分离的设计推导 | First Principles |
| 双周报机制怎么救了项目 | Case-First |
| AI PM 该不该是正式 title | Dialogue |
| 横向复用为什么是 60% 而不是 100% | Case-First + 收口段用 First Principles |

## 反模式

- ❌ 整本书全用同一种模式（读者疲劳）
- ❌ Case-First 没有具体时间地点（变成抽象议论文）
- ❌ Dialogue 一边倒（变成假对话真说教）
- ❌ First Principles 跳步（读者跟丢）
