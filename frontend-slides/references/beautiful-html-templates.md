# Beautiful HTML Templates Reference

Use when building or improving HTML slide decks, sharing pages, dense internal decks, or visual previews where Jianan asks for better aesthetics, nicer color, template inspiration, or less generic AI design.

Source repo:
- Local: `/Users/jianan/.openclaw/workspace/repos/beautiful-html-templates`
- GitHub: `https://github.com/zarazhangrui/beautiful-html-templates`

## What to absorb

Absorb the visual system and template-selection workflow, not the low-density content style.

Useful patterns:
- Template metadata: mood, occasion, tone, formality, density, scheme, best_for, avoid_for.
- Pick by tone and audience, not by industry label alone.
- Generate 2–3 concrete previews before locking visual direction.
- Treat each template as a closed visual system: keep fonts, palette, spacing rhythm, decorative vocabulary, and component grammar consistent.
- If a new slide layout is needed, extend the chosen template; do not mash multiple templates together.

## Jianan-specific rule: density preservation

Jianan's AIPM / CER / internal-sharing decks need higher consulting-style content density than many templates in this repo.

Do:
- Use the templates for color, typography, layout rhythm, and visual polish.
- Preserve rich content: frameworks, process maps, comparison tables, evidence, examples, ROI/risk/timeline logic.
- Split dense content across structured slides rather than deleting substance.

Do not:
- Reduce content into empty slogan slides.
- Copy low-density title/concept-slide style when the task needs deep explanation.
- Use dark neon / purple cyberpunk styles for internal CER/PLO sharing unless explicitly requested.

## Good candidates for Jianan's current work

- `blue-professional`: B2B internal sharing, polished and trustworthy.
