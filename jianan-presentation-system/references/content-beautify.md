# Layer 1: Content Beautification & Script Optimization

接收用户提供的 MD 草稿或口述框架，输出结构更清晰、语言更精炼的版本。支持中文优化、英文翻译、双语对齐三种模式。

---

## 输入格式识别

| 输入类型 | 识别特征 | 处理路径 |
|---------|---------|---------|
| 演讲稿 MD | 有完整段落，口语化，可能有口头停顿词 | → 结构诊断 → 压缩 → 分页映射 |
| 逐点草稿 | 纯 bullet points，无展开 | → 补充论证 → 层级整理 → 金句提炼 |
| 口述框架 | 用户直接在对话中描述思路 | → 整理成大纲 → 逐 slide 展开 |
| 混合文档 | 数据 + 文字混排 | → 分离数据点 → 重构叙事 |

## 三种模式

### 模式 A：中文优化

**目标**：保留原意，提升专业感和简洁度。

优化规则：
1. **删冗余**：去掉「就是说」「也就是」「其实」「相当于」等口头语
2. **精标题**：每个 slide 标题改为"结论句"而非"话题句"（麦肯锡原则）
   - ❌ 「技术架构」
   - ✅ 「三层架构支撑毫秒级响应」
3. **强数字**：凡有数字，前置展示（不要埋在句子中间）
   - ❌ 「准确率有大概 85% 左右」
   - ✅ 「**85% 准确率**，超行业均值 20pt」
4. **对仗化**：并列结构尽量对仗，字数一致
5. **层级清晰**：一页只有一个核心观点，Supporting points ≤ 4 条

### 模式 B：英文翻译

**目标**：自然商务英文，不是直译。

翻译规则：
1. 中文四字词语 → 英文动词短语（「智能识别」→「AI-powered recognition」）
2. 被动句 → 主动句（「该方案已被部署」→「We deployed the solution」）
3. 数字单位注意转换（亿 → 100M / billion，万 → 10K）
4. 中文俗语 / 成语 → 对应英文表达或意译（不硬译）
5. 标题保持 Title Case；bullet points 保持句首大写

### 模式 C：双语对齐输出

格式规范：
```markdown
## Slide N：[中文标题]
**[English Title]**

- 中文要点 1
  *English bullet 1*
- 中文要点 2
  *English bullet 2*

> 演讲稿：[中文演讲稿段落]
> Script: [English script paragraph]
```

## 内容诊断框架

收到草稿后，先输出诊断报告再开始优化：

```markdown
### 内容诊断

| 维度 | 当前状态 | 建议 |
|------|---------|------|
| 叙事结构 | 线性叙事 / 问题-解决 / 数据驱动 | [评估] |
| 信息密度 | 偏多 / 适中 / 偏少 | [评估] |
| 缺失内容 | 无结论 / 无数据支撑 / 无 call-to-action | [列出] |
| 冗余内容 | [具体指出哪些可删] | |
| 亮点保留 | [确认哪些金句要保留原味] | |
```

## 从演讲稿到 Slide 文案的压缩规则

**演讲稿 → Slide 标题**：取核心结论，≤ 15 字
**演讲稿 → Slide Bullets**：每条 ≤ 20 字，去掉所有连接词
**演讲稿 → KPI 数字**：提取可视化的关键数字，放入 KPI 卡
**演讲稿保留**：作为 Speaker Notes 嵌入 PPTX（pptxgenjs 的 `addNotes()`）

示例转换：
```
演讲稿原文：
"在过去三个月的测试期间，我们的系统在处理真实客户的报告对比任务时，
平均准确率达到了 85%，相比之前人工操作的方式大概提升了 40% 左右，
而且处理速度也快了大概三倍。"

→ Slide 标题：  AI比对准确率 85%，速度提升 3×
→ KPI 卡片：    85% | 3× | 40%↑
→ Bullet：      ● 真实业务测试验证，非实验室数据
→ Speaker Note：[保留完整演讲稿原文]
```

## GitHub / 代码库 → 内容提炼

当用户提供 GitHub 链接或本地代码库路径时：

1. **读取顺序**：README → 目录结构 → 核心模块文件（按 import 关系）
2. **提炼维度**：
   - 系统架构：几层？每层职责？
   - 核心流程：数据怎么流动？
   - 关键技术：用了什么框架/模型/算法？
   - 亮点功能：有没有创新点？
3. **输出形式**：
   - 文字摘要（供 Layer 1 内容优化用）
   - 架构列表（供 Layer 3 动图内容用）
