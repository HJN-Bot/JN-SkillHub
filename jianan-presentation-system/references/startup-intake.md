# Startup Intake — Presentation Session 起始交互层

每次启动新 Presentation 项目时，先完成这一层的信息收集，再进入三层工作流。

---

## 起始问卷（Session Kickoff Checklist）

当用户启动新 Presentation 项目时，**必须主动问以下问题**（未提供的项才问，已提供的跳过）：

### 必问项（缺一不可）

```
1. 汇报对象是谁？
   → 高管/董事会 / 客户/甲方 / 内部团队 / 技术评审 / 公开演讲
   → 中文受众 / 英文受众 / 双语

2. 这次 Presentation 的核心目的？
   → 立项审批 / 项目进度汇报 / 方案售前 / 成果展示 / 知识分享

3. 大概多少页 + 多长时间？
   → 影响信息密度和叙事节奏

4. 有没有参考文档？
   → 上次 PPT / 竞品案例 / 产品文档 / GitHub 链接 / 数据报告
```

### 可选项（有就问，无则跳过）

```
5. 有没有特别想强调的模块或结论？
6. 有没有特别不想出现在 PPT 里的内容？
7. 演讲者是自己讲还是给别人讲？（影响是否需要演讲稿）
8. 这次需要哪几层能力？（内容优化 / PPT 设计 / 动图生成）
```

---

## 参考文档接收规则

当用户提供参考资料时，按以下规则处理：

| 资料类型 | 处理方式 |
|---------|---------|
| 历史 PPT（图片/PDF） | 逐页阅读，提炼：叙事结构 + 已有数据 + 视觉风格偏好 |
| GitHub 代码库链接 | 阅读 README + 核心文件，提炼：系统架构 + 模块关系 + 关键功能 → 映射成可视化内容 |
| MD / Word 文稿 | 识别：标题层级 / 核心观点 / 数据点 / 已有金句 |
| 竞品案例图片 | 提炼：布局模式 + 色彩倾向（不复制颜色，只学结构）|
| 数据报告 / 表格 | 提取关键指标，确认哪些进入 KPI 卡片 |

---

## 起始层输出物

完成 Kickoff 后，**在开始任何 Layer 工作前输出一份会话摘要**：

```markdown
## Session Brief

- **汇报对象**: [受众描述]
- **核心目的**: [一句话]
- **规模**: [X 页 / Y 分钟]
- **语言**: [中文 / 英文 / 双语]
- **本次启用的能力**: Layer 1 内容优化 / Layer 2 PPT 设计 / Layer 3 动图
- **参考文档已读**: [列出]
- **叙事主线**: [一句话描述本次 PPT 的核心论点]
- **待确认项**: [列出还不清楚的地方]
```

等用户确认 Session Brief 后，才进入 Layer 工作流。

---

## 触发词

以下情况自动启动起始交互层：
- 「帮我做一个 PPT」「我要做汇报」「我需要一个演示文稿」
- 「这次给 [某人] 讲」「汇报给领导 / 客户 / 投资人」
- 「你帮我整理一下这些材料」（附带文档）
- 「参考上次的 PPT，帮我做...」
