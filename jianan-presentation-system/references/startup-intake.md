# Startup Intake — Presentation Session 起始交互层

每次启动新 Presentation 项目时，先完成这一层的信息收集，再进入三层工作流。

---

## 起始问卷（Session Kickoff Checklist）

当用户启动新 Presentation 项目时，**必须主动问以下问题**（未提供的项才问，已提供的跳过）：

### 战略问题（最重要，必问）

```
1. 听完这个演示，你希望对方做出什么具体决定或行动？
   → 这是整个 PPT 的"北极星"。所有内容都服务于这个决定。
   → 示例：批准立项 / 同意追加预算 / 决定采购 / 理解并支持方案

2. 对方最可能提出的反对意见或顾虑是什么？
   → 这决定哪些 slide 需要专门设计来"消除疑虑"。
   → 示例：成本太高 / 技术不成熟 / 不符合现有流程

3. 你手上最有力的数据或证据是什么？
   → 这是 PPT 的"核弹"，要放在最显眼的位置。
   → 示例：85% 准确率 / 节省 3 倍时间 / 已有成功案例
```

### 信息问题（必问）

```
4. 汇报对象是谁？
   → 高管/董事会 / 客户/甲方 / 内部团队 / 技术评审
   → 中文受众 / 英文受众 / 双语

5. 大概多少页 + 多长时间？

6. 有没有参考文档？
   → 上次 PPT / 产品文档 / GitHub 链接 / 数据报告
```

### 环境问题（影响 Layer 3 能力）

```
7. 需要动图 GIF 吗？（如果需要，问以下）
   → 有没有安装 Node.js？（HTML/CSS 动效用 Puppeteer 录制）
   → 有没有安装 Manim？（3D 架构动画，没有就跳过，改用 HTML/CSS）
   → 有没有安装 FFmpeg？（GIF 转换用）

如果没有相关环境：
   → HTML/CSS 动图：只需浏览器 + QuickTime 录屏（macOS 自带）
   → Manim：跳过，不强制使用
```

### 可选项

```
8. 有没有特别想强调的模块？
9. 演讲者是自己讲还是给别人讲？（影响是否需要演讲稿）
10. 需要哪几层能力？（内容优化 / PPT 设计 / 动图）
```

---

## 参考文档接收规则

当用户提供参考资料时，按以下规则处理：

| 资料类型 | 处理方式 |
|---------|---------|
| 历史 PPT（图片/PDF） | 逐页阅读，提炼：叙事结构 + 已有数据 + 视觉风格偏好 |
| GitHub 代码库链接 | 阅读 README + 核心文件，提炼：系统架构 + 模块关系 + 关键功能 → 映射成可视化内容 |
| MD / Word 文稿 | 识别：标题层级 / 核心观点 / 数据点 / 已有金句 |
| 竞品案例图片 | 提炼：布局模式 + 色彩倾向（不复制颜色，只学结构）|
| 数据报告 / 表格 | 提取关键指标，确认哪些进入 KPI 卡片 |

## 起始层输出物

完成 Kickoff 后，**在开始任何 Layer 工作前输出一份会话摘要**：

```markdown
## Session Brief

- **汇报对象**: [受众描述]
- **目标决定**: [听完后对方需要做的具体决定]
- **核心证据**: [最强数据 / 案例]
- **主要顾虑**: [对方可能的反对意见 → 对应 slide 策略]
- **规模**: [X 页 / Y 分钟]
- **语言**: [中文 / 英文 / 双语]
- **启用能力**: Layer 1 / Layer 2 / Layer 3（有 Manim ✓/✗ | 有 FFmpeg ✓/✗）
- **参考文档**: [列出已读的]
- **叙事主线**: [一句话：因为[证据]，所以你应该[决定]]
- **待确认**: [还不清楚的地方]
```

等用户确认 Session Brief 后，才进入 Layer 工作流。

---

## 触发词

以下情况自动启动起始交互层：
- 「帮我做一个 PPT」「我要做汇报」「我需要一个演示文稿」
- 「这次给 [某人] 讲」「汇报给领导 / 客户 / 投资人」
- 「你帮我整理一下这些材料」（附带文档）
- 「参考上次的 PPT，帮我做...」
