# Impeccable Design 技能

基于 impeccable.style 的设计原则和模式，提供专业的前端设计指导。

## 核心设计原则

### 1. 排版设计 (Typography)
- **字体选择**：优先使用系统字体栈，确保跨平台一致性
- **字体大小**：使用固定的字体比例尺（如 1.25 或 1.333 的比例）
- **行高**：正文行高 1.5-1.75，标题行高 1.2-1.3
- **字重**：合理使用字重层次（常规、中等、粗体）
- **对齐**：左对齐为主，避免居中对齐长文本

### 2. 色彩系统 (Color System)
- **主色调**：选择 1-2 个主色，用于主要操作和品牌元素
- **中性色**：建立完整的灰度系统（从浅灰到深黑）
- **语义色**：成功（绿）、警告（黄）、错误（红）、信息（蓝）
- **对比度**：确保文本与背景的对比度符合 WCAG AA 标准（4.5:1）

### 3. 布局与间距 (Layout & Spacing)
- **间距系统**：使用 4px 或 8px 的倍数作为间距单位
- **容器宽度**：主要内容区域最大宽度 1200px
- **响应式断点**：移动端（<768px）、平板（768-1024px）、桌面（>1024px）
- **网格系统**：使用 CSS Grid 或 Flexbox 创建灵活的布局

### 4. 交互与动效 (Interaction & Motion)
- **微交互**：按钮悬停、焦点状态、加载状态
- **过渡动画**：使用 CSS transition 实现平滑的状态变化
- **持续时间**：交互动画 150-300ms，页面过渡 300-500ms
- **缓动函数**：使用 ease-out 或 cubic-bezier(0.4, 0, 0.2, 1)

### 5. 组件设计 (Component Design)
- **一致性**：相同功能的组件保持相同的外观和行为
- **状态管理**：考虑所有可能的状态（默认、悬停、焦点、禁用、加载、错误）
- **可访问性**：支持键盘导航和屏幕阅读器
- **响应式**：组件在不同屏幕尺寸下都能正常工作

## 设计命令参考

### /polish - 优化现有设计
检查并改进现有设计的以下方面：
1. 排版层次和可读性
2. 色彩对比度和可访问性
3. 间距一致性和视觉节奏
4. 交互反馈和用户体验

### /audit - 设计审计
系统性地检查设计问题：
1. 可访问性违规
2. 响应式问题
3. 性能影响
4. 代码质量

### /distill - 简化设计
去除不必要的元素，专注于核心功能：
1. 移除装饰性元素
2. 简化颜色方案
3. 减少字体变化
4. 优化布局结构

### /bolder - 增强视觉冲击力
让设计更加突出：
1. 增加对比度
2. 使用更大胆的字体
3. 添加微妙的阴影或边框
4. 优化焦点状态

### /typeset - 排版优化
专门优化排版：
1. 调整字体大小比例
2. 优化行高和字间距
3. 改善段落间距
4. 确保标题层次清晰

### /arrange - 布局优化
优化布局和间距：
1. 建立一致的间距系统
2. 优化网格对齐
3. 改善视觉层次
4. 确保响应式布局

### /overdrive - 高级效果（Beta）
添加技术性特效：
1. 视差滚动
2. 粒子效果
3. 3D变换
4. 高级动画

## 常见设计模式

### 技术文档风格
- 清晰的代码示例展示
- 步骤化的安装指南
- 版本更新说明
- 故障排除指南

### 卡片式设计
- 内容模块化展示
- 一致的卡片尺寸和间距
- 悬停和焦点状态
- 响应式排列

### 渐进式披露
- 复杂信息分层展示
- 可展开/折叠的内容区域
- 工具提示和弹出框
- 模态对话框

### 数据可视化
- 清晰的图表和图形
- 交互式数据探索
- 响应式图表
- 可访问的颜色方案

## 反模式避免

### 排版反模式
- ❌ 使用太多不同的字体
- ❌ 行高过小或过大
- ❌ 居中对齐长段落
- ❌ 使用纯黑色文字在纯白色背景上

### 色彩反模式
- ❌ 使用太多鲜艳的颜色
- ❌ 低对比度的文本
- ❌ 不一致的颜色使用
- ❌ 忽略色盲用户的需求

### 布局反模式
- ❌ 不一致的间距
- ❌ 过于复杂的网格
- ❌ 忽略移动端体验
- ❌ 内容区域过宽

### 交互反模式
- ❌ 过长的动画持续时间
- ❌ 缺少焦点状态
- ❌ 不可预测的交互行为
- ❌ 忽略键盘导航
