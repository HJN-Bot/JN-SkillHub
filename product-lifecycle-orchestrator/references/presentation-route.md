# PLO Presentation Route

PLO owns lifecycle routing. PVO owns presentation expression.

## Trigger

If the T2+ deliverable includes any of these, route through PVO:

- PPT / slide deck / HTML deck
- public sharing / internal sharing
- boss brief / executive update / customer deck
- product demo / launch / roadshow
- visual report / social cover / image-heavy presentation
- screenshot-heavy proof deck

## Boundary

PLO decides:

- project mode and phase;
- stakeholder / decision context;
- whether the work is T2+;
- durable project state;
- delivery evidence and follow-up.

PVO decides:

- audience belief shift;
- presentation type;
- output engine: HTML deck / native PPTX / image route / outline;
- style lock;
- page plan;
- image manifest;
- visual QA.

## Routing table

| Deliverable | PLO path | PVO route |
|---|---|---|
| Boss decision deck | B2B Content/Influence | PVO + decision deck + native PPTX if editability matters |
| Internal/public sharing | B2B Content or Personal/Internal | PVO + HTML deck or PPTX based on venue |
| Product demo deck | B2B/ToC depending product | PVO + product promotion / wow route |
| Architecture/codebase deck | B2B Enterprise or Internal | PVO + hard-core analysis route |
| Training/knowledge sharing | B2B Content or Personal | PVO + teaching route |
| Cover/social visual | Content/Influence | PVO image/cover route |

## Project State addition

For presentation projects, Project State should include:

- audience;
- desired belief shift;
- output format;
- deck type;
- style family;
- proof assets;
- image/screenshot needs;
- current PVO artifacts;
- visual QA status.
