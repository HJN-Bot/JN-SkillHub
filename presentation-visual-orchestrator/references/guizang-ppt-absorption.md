# Guizang PPT Absorption Notes

Absorb the principles; do not blindly copy the third-party skill.

## Useful ideas

1. Two strong visual systems:
   - Editorial magazine / electronic ink: story, opinion, human texture, public sharing.
   - Swiss international style: structure, data, product, engineering, method, analysis.
2. Start with style, audience, duration, source material, image/screenshot needs, theme color, hard constraints.
3. Use fixed layout systems for quality. A style is not only colors; it is page structure, grid, typography, image slots, and QA.
4. Swiss-style work benefits from locked layouts: choose known page types rather than inventing every slide.
5. Image generation is optional and must be asked/confirmed for meaningful cost or external generation.
6. Generate images for slots, not for standalone beauty.

## PVO adaptations

- Ask at most 1–3 blocking questions, not a rigid 7-question interview.
- Create `STYLE_LOCK.md` and `PAGE_PLAN.md` before HTML/PPT execution.
- If Swiss/structured style is selected, require layout names and image slots per page.
- For screenshots: preservation first, redesign second.
- For generated information graphics: match deck language and theme.
- For deck images: no title/page/footer/decoration inside the image itself.

## Image types to support

- documentary/human photo
- information graphic
- flow / comparison / system relationship diagram
- screenshot framing / screenshot redesign
- data poster / big number visual
- social cover / share card

## QA rules worth keeping

- no generic AI aesthetic;
- no emoji as design icons in serious decks;
- text must remain readable;
- title hierarchy must be obvious;
- images must be legible and aligned;
- theme rhythm should breathe across pages;
- generated visuals must not repeat slide chrome.
